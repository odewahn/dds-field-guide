The Distributed Developer Stack Field Guide

This site tracks the key theme and tools in an emerging trend we're calling the "Distributed Development Stack" (DDS). The DDS stack covers DevOps, software engineering, programming practice, organizational culture, and a host of other factors.

How to contribute

  1. Agree to the O'Reilly contributor license agreement
  2. Fork the repo
  3. Send a pull request
  4. Add yourself to the contributor page.

Key themes we're exploring...

The cloud is the default platform.
CI servers deploy code, not ops.
The codebase is in git.
The application runs locally on development.
The environment is automated in the code.
The monitoring infrastructure is critical.
Tests done in code, not by a QA department.
